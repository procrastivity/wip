## Round 3 ŌĆö Porcelain, plugin & features

- **step-10 ŌĆö `wip` porcelain** Ō£ģ shipped 2026-06-13 ŌĆö `bin/wip` ships with `ask` (single-turn chat completion) + `provider show` (config diagnostic, never prints the api_key); every other verb is `exec`'d through to `bin/wip-plumbing` byte-identically. Provider config reads `.wip.yaml`'s `provider:` block; v1 supports `kind: openai-compatible` only. Test seam: `WIP_PROVIDER_CMD` replaces curl with a shell snippet fed the request JSON on stdin, so `make check` stays network-free. Error envelope kinds (`no-provider` / `bad-provider` / `unsupported-provider` / `provider-env-unset`) include payload fields (`error.env`, `error.provider_kind`) so step-10.5's shaper can branch on them. Adds `curl` to `flake.nix` + Makefile `deps-check`. Spec: [`engineering/specs/wip-porcelain.md`](../../../engineering/specs/wip-porcelain.md).
- **step-10.5 ŌĆö `wip intake` porcelain** Ō£ģ shipped 2026-06-13 ŌĆö `wip intake <file>` drives the full intake pipeline end-to-end: classify (shellout) ŌåÆ shape (LLM via the step-10 provider seam) ŌåÆ validate (shellout) ŌåÆ route ŌåÆ apply. The shaper emits clarifying questions via a `---ASK---` fenced block the porcelain parses; the user answers on stdin/`/dev/tty`. Flags: `--kind` / `--target` / `--yes` / `--dry-run` / `--output` / `--max-rounds`. Stdout envelope counts every LLM call under `rounds` (initial + ASK turns + validate retries) and records each clarifying question under `asked[]`. New error kinds: `classify-failed`, `kind-ambiguous`, `shape-failed`, `ask-without-tty`, `bad-shape-response`, `apply-failed`. New `lib/wip/wip-intake-shaper-lib.bash` inlines the per-kind shape rules from `intake-kinds.md` ┬¦2/┬¦3. Tests reuse step-10's `WIP_PROVIDER_CMD` seam, stateful counter-files cover retry / ASK / round 2 conversation shape. Dogfood: round-tripped `~/.claude/plans/explore-how-clast-whimsical-floyd.md` into a fresh `intake-pipeline` roadmap as `step-03 ŌĆö porcelain shaper` (mocked provider per `WIP_PROVIDER_CMD` since no live key in the dev shell; transcripts under `.wip/scratch/dogfood-step-10.5-*`). Porcelain version bumped to `0.2.0-dev`.
