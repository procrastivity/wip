## Round 3 — Porcelain, plugin & features  ✅ shipped 2026-06-14

- **step-10 — `wip` porcelain** ✅ shipped 2026-06-13 — `bin/wip` ships with `ask` (single-turn chat completion) + `provider show` (config diagnostic, never prints the api_key); every other verb is `exec`'d through to `bin/wip-plumbing` byte-identically. Provider config reads `.wip.yaml`'s `provider:` block; v1 supports `kind: openai-compatible` only. Test seam: `WIP_PROVIDER_CMD` replaces curl with a shell snippet fed the request JSON on stdin, so `make check` stays network-free. Error envelope kinds (`no-provider` / `bad-provider` / `unsupported-provider` / `provider-env-unset`) include payload fields (`error.env`, `error.provider_kind`) so step-10.5's shaper can branch on them. Adds `curl` to `flake.nix` + Makefile `deps-check`. Spec: [`engineering/specs/wip-porcelain.md`](../../../engineering/specs/wip-porcelain.md).
- **step-10.5 — `wip intake` porcelain** ✅ shipped 2026-06-13 — `wip intake <file>` drives the full intake pipeline end-to-end: classify (shellout) → shape (LLM via the step-10 provider seam) → validate (shellout) → route → apply. The shaper emits clarifying questions via a `---ASK---` fenced block the porcelain parses; the user answers on stdin/`/dev/tty`. Flags: `--kind` / `--target` / `--yes` / `--dry-run` / `--output` / `--max-rounds`. Stdout envelope counts every LLM call under `rounds` (initial + ASK turns + validate retries) and records each clarifying question under `asked[]`. New error kinds: `classify-failed`, `kind-ambiguous`, `shape-failed`, `ask-without-tty`, `bad-shape-response`, `apply-failed`. New `lib/wip/wip-intake-shaper-lib.bash` inlines the per-kind shape rules from `intake-kinds.md` §2/§3. Tests reuse step-10's `WIP_PROVIDER_CMD` seam, stateful counter-files cover retry / ASK / round 2 conversation shape. Dogfood: round-tripped `~/.claude/plans/explore-how-clast-whimsical-floyd.md` into a fresh `intake-pipeline` roadmap as `step-03 — porcelain shaper` (mocked provider per `WIP_PROVIDER_CMD` since no live key in the dev shell; transcripts under `.wip/scratch/dogfood-step-10.5-*`). Porcelain version bumped to `0.2.0-dev`.
- **step-11 — `/wip:*` plugin** ✅ shipped 2026-06-13 — `.claude-plugin/` with `plugin.json` (name=`wip`) + `commands/{next,status,intake}.md` user-invoked slash commands resolving as `/wip:next`, `/wip:status`, `/wip:intake`. Each shells out to `bin/wip-plumbing`; Claude Code is the LLM (no provider call). `/wip:intake` drives the full intake pipeline from inside a Claude Code session with **Claude as the shaper** — clarifying questions happen inline in chat (no `---ASK---` fence). Load-bearing seam: shaper prompts lifted from heredocs in `lib/wip/wip-intake-shaper-lib.bash` to `templates/prompts/intake/{preamble,brief,amendment,workplan-seed,spec,handoff}.md`; the lib reads them from disk; the plugin reads them via the new `wip-plumbing template show <id>` verb so plugin command bodies say *what they want* not *where the bytes live*. Equivalence pinned by `test/test-shaper-templates.sh` (verb-show byte-equiv against source files + lib's `wip_shaper_system_prompt` contains both halves verbatim). Adds the `template show|list` plumbing verb (doc'd in `wip-plumbing-cli.md` §3), the `WIP_TEMPLATES_DIR` env override (test seam + install seam), and a new spec [`engineering/specs/wip-plugin.md`](../../../engineering/specs/wip-plugin.md). Three new test suites (`test-shaper-templates.sh`, `test-template-verb.sh`, `test-plugin-manifest.sh`) — 55 new assertions, all 22 prior suites still green, doctor clean. `agents/` is a 3-line README stub pointing at `roles/` (files land step-12). Dogfood: `/wip:next` + `/wip:status` + `/wip:intake` executed against this repo from inside Claude Code; full transcripts (including the inbound, shaped artifact, apply ledger, roadmap diff + rollback) at `.wip/scratch/dogfood-step-11-*`.
- **step-12 — Roles set** ✅ shipped 2026-06-13 — distilled `workflow-portable-stub/playbook/` (6 markdown files) into the backend-agnostic `roles/` per [`roles/README.md`](../../../roles/README.md): behavior files (`orchestrator.md` / `coordinator.md` / `researcher.md` / `builder.md` / `shared.md`) + `tier-policy.md` + `backends/solo.md` (the only doc naming Solo MCP tools — `mcp__solo__*`, `agent_tool_id`, `list_agent_tools`, `whoami`, `mcp-cli` anti-pattern). Plugin `.claude-plugin/agents/{orchestrator,coordinator,researcher,builder}.md` ship as thin `wip-<role>` agent definitions whose bodies are one sentence of framing + `@`-file pointers into `roles/` (single source of truth — same prompt-sharing seam shape as step-11's templates). ADR-0007's acceptance shape pinned by `test/test-roles-backend-seam.sh` (60 assertions): zero Solo-token matches in behavior + `tier-policy`, expected matches in `backends/solo.md`, all plugin `@`-file references resolve. `test-plugin-manifest.sh` updated to assert the four agent files. Roadmap stays — `wip spawn` / `wip orchestrate` (w4/w5) are still future verbs that will use the Tier policy. (was scratchpad item 2.)
- **step-13 — `wip glossary` assembler** ✅ shipped 2026-06-13 — `bin/wip-plumbing glossary assemble` renders `core.md` + enabled-feature partials in declaration order (inclusion-rule table is single source of truth in `lib/wip/wip-plumbing-glossary-lib.bash`; `solo.md` correctly gated on `features.orchestration.enabled && features.orchestration.backend == "solo"` per ADR-0007). Stdout-first (`--output <path>` for atomic write + JSON ledger). `glossary check` exits 4 on drift (content / manifest / partial), wired into pre-commit as `wip-glossary` local hook so a stale glossary fails CI. Partial-author HTML comment headers are stripped on emit; assembled file carries one generated header (Source paths + Driven-by predicates + Regenerate/Verify recipes). Predicate-true-but-partial-not-shipped (e.g. `diataxis.md` here) is a graceful skip with ledger entry — `lds.md` / `diataxis.md` partials land as one-row additions to the rule table with zero code change. Lifted `_wip_template_dir` → `wip_templates_dir` to the shared lib (two consumers now, three when LDS lands). Bootstrap placeholder at `.wip/GLOSSARY.md` replaced with the real concatenated artifact; `make glossary` regen target added.
- **step-14 — `wip setup` family** ✅ shipped 2026-06-14 — `bin/wip-plumbing setup {deps,direnv,hygiene,release,agents}` — five plumbing verbs that vendor `templates/setup/<capability>/` bytes into a consumer repo (verbatim, no `{{key}}` substitution) and flip the matching `features.<name>.enabled` flag (direnv → `features.direnv`; release → `features.changelog`; agents → `features.orchestration.{enabled, backend: solo, source: plugin}` per ADR-0007; deps + hygiene track nothing). Three-way idempotency per file (absent → write; byte-equal → silent skip; differs → exit 4 `content-drift` unless `--force`); `flake.lock` is special-cased to "skip if present, never compare" since locks evolve per consumer. Composition over chaining: `setup direnv` exits 3 `missing-prereq` when `flake.nix` is absent, with a hint to run `setup deps` first; all verbs exit 3 `missing-manifest` when `.wip.yaml` is absent. Sentinel post-check drives the invariant that `doctor` sees zero drift for any flipped feature. `templates/setup/agents/.claude-plugin/` is the live plugin tree with `bin/wip-plumbing` → `wip-plumbing` substituted at template-author time (consumers expect wip on PATH). Templates under `deps/direnv/hygiene/` are byte-derived `cp`s from this repo's step-09 files, asserted by `test/test-setup.sh`'s `cmp` cases (74 assertions; full tempdir round-trip dogfood: `init` + five verbs + `cmp` of every output to the live equivalent, plus `doctor drift=0`). New `lib/wip/wip-plumbing-setup-lib.bash` (helpers) + `lib/wip/wip-plumbing-subcommands/setup.bash` (dispatcher) + spec section in `wip-plumbing-cli.md`. Existing tests all green (still 23 prior suites + 1 new); pre-commit clean.
- **step-15 — `graduate` / `extract`** ✅ shipped 2026-06-14 — Two top-level plumbing verbs ship the LDS seam per ADR-0006. `graduate <artifact>` promotes one wip-internal planning artifact (read its `graduate-to:` front-matter directive, or pass `--to <slot>`) to its canonical LDS slot under `<eng-docs>/<layer>/<file>.md`, three-way idempotent on the bytes (reuses `wip_setup_write_idempotent`). `decisions/auto-<slug>.md` shorthand resolves to the next 4-digit ADR number. Layer allowlist enforced (the 7 canonical LDS layers + `maintenance` + `appendices`). `extract [--manifest <p>]` runs the deterministic LDS Extract phase against an approved extraction manifest (default `<eng-docs>/.lds-manifest.yaml`) — v1 supports `verbatim` + `content` modes and simple-path / single-file-with-range source specs; `transform` / `summarize` / multi-file / `template+field_mappings` land in `unsupported[]` (skip, don't fail). SHA-256 hash verification is parsed-but-skipped in v1 (ledger flag `hash_verification: "skipped-v1"`). Source attribution comments per LDS §6.3 are mandatory (part of the idempotency bytes). New `lib/wip/wip-plumbing-{graduate,extract}-lib.bash` + subcommand dispatchers; lifted `wip_json_string_array` to the shared setup lib for reuse. `test/test-graduate.sh` (35 assertions) + `test/test-extract.sh` (51 assertions); all 25 prior suites still green. Dogfood: tempdir consumer with `features.lds.enabled: true`, graduated a step-13 decision block into `engineering/decisions/0001-glossary-verb-shape.md` and re-ran to verify idempotency. **Round 3 closes here**; 9 follow-up items captured in §Backlog (transform/summarize modes, hash verification, multi-file, templates, resume, extraction report, `setup lds`, glossary partial, intake-apply spec dispatch).

## Backlog (cross-cutting; see also `.wip/backlog.md`)

- **In-place study-slice fixes** (scratchpad item 3): fix `prtend/CLAUDE.md` (→ xcind
  pointer) and `workflow-portable-stub` broken paths *in the gitignored slices*. Needs a
  human call — these are reference copies and prtend is a useful counter-example.
- **extract-transform-mode** — LDS `transform` mode (heading_adjust / link_rewrite / markdown_format).
  v1 routes these to `unsupported[]`. Requires a small markdown engine in bash + per-transform options; ship when a consumer needs it.
- **extract-summarize-mode** — LDS `summarize` mode. Inherently LLM-driven (LDS itself says "NEVER automatic"); belongs in porcelain, not plumbing. v1 routes to `unsupported[]`.
- **extract-verify-hashes** — `--verify-hashes` flag to enable SHA-256 source hash verification. Requires `sha256sum`/`shasum` in the flake (add to `setup deps` template). v1 ledger advertises `hash_verification: "skipped-v1"`.
- **extract-multi-file-source** — Multi-file source specs (`source.files[]` with separator + `combined_hash`). v1 routes to `unsupported[]`. Add when a real manifest needs concatenation.
- **extract-templates-field-mappings** — LDS template + `field_mappings` support per `extract.md` §4 (literal + `source:<path>:<lines>` references). v1 skips templated entries. Add when a consumer adopts MADR/PRD-Lite templates with field maps.
- **extract-resume-mode** — `--resume` flag for partial-failure recovery (skip-if-exists per entry). Three-way idempotency already gives re-runs a sensible shape; full resume is a separate contract.
- **extract-extraction-report** — Write `extraction-report.{md,yaml}` to disk per LDS §7 (metadata + summary + per-entry rows). v1 ledger is stdout-only.
- **setup-lds-verb** — Sixth `setup` verb to install the LDS scaffold (manifest skeleton, layer directories, `maintenance/` workflow copies) and flip `features.lds.enabled`. The hint in `lds-not-enabled` / `lds-sentinel-missing` errors from `graduate`/`extract` points at this item.
- **glossary-partial-lds** — Author `templates/glossary/lds.md`. The inclusion rule is already declared in `wip-plumbing-glossary-lib.bash` (predicate `features.lds.enabled == true`) with graceful skip when the partial file is absent; this just ships the partial bytes.
- **intake-apply-spec-graduate-dispatch** — Route `intake apply --kind spec` through to `graduate` when the artifact carries a `graduate-to:` directive (currently still exits 3 stub). Needs the porcelain spec-shaper to insert the directive — porcelain change, not plumbing.
