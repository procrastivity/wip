# docs/ — User documentation (Diátaxis)

Documentation for people **consuming `wip`** in their own projects. Organized by the
four Diátaxis modes. Independent of [`../engineering/`](../engineering/) (which is the
*internal* canon for building `wip`).

| Mode | Path | Holds |
|------|------|-------|
| Tutorial | `getting-started/` | learning-oriented: install, first initiative |
| How-to | `guides/` | task-oriented: enable a feature, graduate to LDS, orchestrate with Solo |
| Reference | `reference/` | information-oriented: `wip` / `wip-plumbing` CLI, `.wip.yaml` schema |
| Explanation | `explanation/` | understanding-oriented: what is `wip`, the plumbing/porcelain split, composability |

> 🚧 Stubs only until the CLI exists — content is written against shipped behavior, not
> design intent. The vocabulary these docs use comes from the assembled glossary
> (`core` + enabled-feature partials), never from the internal Solo-laden version.
